# 이 폴더의 작업 규칙

## 폴더 소개

여기는 **개발 연습용 작업 폴더**입니다.

- **Next.js** — 웹사이트를 만드는 도구
- **Supabase** — 데이터를 저장하는 온라인 데이터베이스
- **Vercel** — 만든 웹사이트를 인터넷에 올리는 배포 서비스

---

## Claude의 행동 규칙

### 1. 설명은 쉽게
- 개발 용어는 최대한 **쉬운 말로 풀어서** 설명한다
- 예: "배포" → "인터넷에 올리기", "마이그레이션" → "데이터베이스 구조 변경", "컴포넌트" → "화면 조각"
- 전문 용어를 쓸 때는 괄호 안에 쉬운 설명을 함께 쓴다

### 2. 진행 상황을 자주 말하기
- 작업을 시작할 때, 진행 중일 때, 완료했을 때 각각 **지금 무엇을 하고 있는지** 알린다
- 예: "지금 로그인 화면을 만들고 있어요", "데이터베이스에 연결하는 중이에요", "완료됐어요!"

### 3. 폴더 생성 금지
- 작업폴더 루트(`ggm/`)에 **새로운 폴더를 만들지 않는다**
- 기존 폴더(`goguma-market/` 등) 안에 하위 폴더를 만드는 것은 허용

### 4. 지시한 것만 단계적으로
- 사용자가 말한 것 **딱 그것만** 한다 — 요청하지 않은 기능 추가, 리팩토링, 구조 변경 금지
- 한 번에 한 단계씩, 완료 후 다음 단계로 넘어간다
- 다음에 할 일이 있어도 먼저 허락을 받고 진행한다

### 5. 할 수 있는 건 최대한 직접 처리
- 파일 생성, 코드 작성, 설정 변경 등 Claude가 할 수 있는 것은 **직접 다 처리**한다
- 사람이 반드시 해야 하는 것(로그인, 결제, 클릭 등)만 따로 안내한다
- 사람이 할 일은 **번호를 붙여 순서대로** 알기 쉽게 안내한다

### 6. 작업 완료 후 바로 확인할 수 있게
- 개발 작업이 끝나면 **항상 바로 결과를 열어서 확인**한다
- 개발 서버를 실행하고 브라우저에서 동작을 확인한 뒤 완료를 알린다
- 화면 변경이 있으면 스크린샷이나 미리보기로 보여준다

## 사람이 직접 해야 하는 것들 (Claude가 대신 못 하는 것)

- Supabase 대시보드에서 직접 설정 변경
- Vercel에 로그인해서 배포 승인
- 환경변수(.env) 파일에 비밀 키 직접 입력
- GitHub에 푸시(push)하기 — 코드를 올리는 행위

이런 작업이 필요하면 **단계별로 스크린샷이나 메뉴 경로와 함께** 안내한다.
